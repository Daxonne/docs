# Getting Started

This guide walks you through your first Daxonne code generation in 5 minutes.

## Prerequisites

- Go 1.23 or later
- A running database (Oracle, PostgreSQL or MySQL)

## Installation

```bash
go install github.com/daxonne/core@latest
```

Or build from source:

```bash
git clone https://github.com/Daxonne/core.git
cd core
go build -o daxonne .
```

---

## Step 1 — Initialise the project

Run `daxonne init` in your project directory. It asks a few questions and creates `daxonne.yaml`:

```bash
daxonne init
```

```
Database type (oracle/postgres/mysql): oracle
Connection string: oracle://myuser:mypass@localhost:1521/FREEPDB1
Schema owner / database name: MYUSER
Output path [./generated]:
```

This creates `daxonne.yaml` (gitignored — never commit credentials):

```yaml
database:
  type: oracle
  connection: "oracle://myuser:mypass@localhost:1521/FREEPDB1"
  owner: "MYUSER"

output:
  path: "./generated"

templates: []
```

:::tip
Copy `daxonne.yaml.example` as a starting point if you prefer to edit manually.
:::

---

## Step 2 — Pull the schema

```bash
daxonne pull
```

Daxonne connects to your database, reads all tables and their columns, primary keys, and foreign keys, then saves the result to `.daxonne/schema.json`.

```
Connecting to oracle database...
Reading schema for owner "MYUSER"...

Schema pulled successfully!
  Tables      : 12
  Columns     : 87
  Foreign keys: 9
  Cached at   : .daxonne/schema.json
```

You can re-run `daxonne pull` any time your schema changes.

---

## Step 3 — Install a template

```bash
daxonne add csharp-dapper
```

Daxonne downloads the template from [github.com/Daxonne/templates](https://github.com/Daxonne/templates) and installs it into `.daxonne/templates/csharp-dapper/`. It also adds the template name to `daxonne.yaml`.

List all available templates:

```bash
daxonne add --list
```

```

python-sqlalchemy         1.0.0      python         Python SQLAlchemy models + async repositories
```

## Step 4 — Generate

```bash
daxonne generate
```

```

  + generated/CustomerDto.cs
  + generated/CustomerRepository.cs
  + generated/OrderDto.cs
  + generated/OrderRepository.cs
  ...

Done! 24 file(s) written to ./generated
```

## What gets generated

For each table in your schema, the `csharp-dapper` template produces:

**`CustomerDto.cs`**
```csharp
// Auto-generated by Daxonne — do not edit manually

public record CustomerDto(
    long Id,
    string FirstName,
    string LastName,
    string? Email,
    decimal Balance,
    DateOnly JoinedAt
);
```

**`CustomerRepository.cs`**
```csharp
// Auto-generated by Daxonne — do not edit manually

using System.Data;
using Dapper;

public class CustomerRepository
{
    private readonly IDbConnection _db;
    public CustomerRepository(IDbConnection db) => _db = db;

    public async Task<IEnumerable<CustomerDto>> GetAllAsync() { ... }
    public async Task<CustomerDto?> GetByIdAsync(long id) { ... }
    public async Task<int> InsertAsync(CustomerDto dto) { ... }
    public async Task<int> DeleteAsync(long id) { ... }
}
```
